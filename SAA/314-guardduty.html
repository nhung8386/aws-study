GuardDuty: Phát hiện mối đe dọa thông minh

**Amazon GuardDuty** giúp bạn thực hiện khám phá mối đe dọa thông minh để bảo vệ các tài khoản AWS của mình. Nó sử dụng các thuật toán **học máy (Machine Learning)**, thực hiện **phát hiện bất thường (Anomaly Detection)**, và sử dụng dữ liệu bên thứ ba.

Kích hoạt GuardDuty chỉ với **một cú nhấp chuột** và bắt đầu ngay với thời gian dùng thử **30 ngày** miễn phí.

Nguồn Dữ Liệu Đầu Vào Được Phân Tích

MANDATORY: Nguồn dữ liệu bắt buộc (Luôn được phân tích)

CloudTrail Event Logs

Giám sát các lệnh gọi API bất thường hoặc triển khai trái phép (Sự kiện quản lý và dữ liệu S3).

VPC Flow Logs

Phân tích lưu lượng truy cập internet bất thường và xác định các địa chỉ IP đáng ngờ.

DNS Logs

Phát hiện các phiên bản EC2 gửi dữ liệu được mã hóa trong các truy vấn DNS (nguy cơ bị xâm nhập).

OPTIONAL: Nguồn dữ liệu tùy chọn (Có thể bật thêm)

EKS Audit Logs

RDS/Aurora Login Events

EBS Volumes

Lambda Network Activity

S3 Data Events

Tự Động Hóa Phản Ứng với Amazon EventBridge

1. GuardDuty Phát hiện Findings (Phát hiện mối đe dọa)
2. Tạo Event trong Amazon EventBridge
3. EventBridge Rules Kích hoạt Hành động Tự động
Target 1: SNS Topic (Thông báo)
Target 2: Lambda Function (Phản ứng tự động: cách ly, chặn...)

Khả năng tự động hóa này là chìa khóa cho phản ứng bảo mật theo thời gian thực (Real-time Security Response).

Điểm Chính Cần Ghi Nhớ

**GuardDuty** cung cấp khám phá mối đe dọa thông minh bằng cách sử dụng **ML** và **phát hiện bất thường**.

Phân tích các nguồn dữ liệu chính: **VPC Flow Logs**, **CloudTrail Logs**, và **DNS Logs**. Các nguồn khác (**EKS, S3, RDS, EBS, Lambda**) là tùy chọn.

Bao gồm các tính năng phát hiện chuyên biệt cho các cuộc tấn công **Tiền điện tử (Cryptocurrency Attacks)**.

Findings tự động tạo **Amazon EventBridge** Events, cho phép các phản ứng tự động thông qua **Lambda** hoặc **SNS**.
