AWS Database Migration Service (DMS)

Dịch vụ Di chuyển Cơ sở Dữ liệu Nhanh chóng và An toàn.

Giới Thiệu về AWS Database Migration Service (DMS)

Để di chuyển cơ sở dữ liệu từ hệ thống tại chỗ (on-premises) của bạn lên AWS Cloud, bạn nên sử dụng DMS, hoặc Database Migration Service. Đây là một dịch vụ cơ sở dữ liệu nhanh chóng và an toàn, cho phép bạn di chuyển cơ sở dữ liệu lên AWS. Dịch vụ này có khả năng phục hồi và tự phục hồi (self-healing). Quan trọng là, trong quá trình di chuyển, cơ sở dữ liệu nguồn vẫn sẵn sàng hoạt động.

Hỗ trợ Nhân bản Dữ liệu Liên tục

DMS hỗ trợ nhân bản dữ liệu liên tục bằng cách sử dụng Change Data Capture (CDC).

Để sử dụng DMS, bạn cần tạo một EC2 instance. Instance này chạy phần mềm DMS và thực hiện các tác vụ nhân bản, liên tục kéo dữ liệu từ nguồn và chèn vào đích.

Các Loại Di chuyển Cơ sở Dữ liệu

Homogeneous Migrations (Đồng nhất)

Di chuyển giữa các công cụ cơ sở dữ liệu giống nhau.

Ví dụ: Oracle sang Oracle hoặc PostgreSQL sang PostgreSQL. Trong trường hợp này, không cần sử dụng Schema Conversion Tool (SCT).

Heterogeneous Migrations (Dị biệt)

Di chuyển giữa các công cụ cơ sở dữ liệu khác nhau.

Ví dụ: Microsoft SQL Server sang Aurora. Trong trường hợp này, cần sử dụng AWS Schema Conversion Tool (SCT).

Sử dụng AWS Schema Conversion Tool (SCT)

Nếu cơ sở dữ liệu nguồn và đích không cùng loại công cụ, bạn phải sử dụng AWS SCT (Schema Conversion Tool).

SCT chịu trách nhiệm chuyển đổi lược đồ cơ sở dữ liệu (schema) từ công cụ này sang công cụ khác (ví dụ: chuyển đổi từ Oracle hoặc SQL Server sang MySQL, PostgreSQL hoặc Aurora).

Hãy nhớ: Di chuyển giữa cùng một công cụ (ví dụ: PostgreSQL tại chỗ sang RDS PostgreSQL) không cần SCT. Di chuyển giữa các công cụ khác nhau (ví dụ: Oracle sang PostgreSQL) cần SCT.

Các Cơ sở Dữ liệu Nguồn và Đích Được Hỗ trợ

Ý tưởng chung là DMS có thể giúp bạn di chuyển cơ sở dữ liệu từ hầu hết mọi nơi đến hầu hết mọi dịch vụ cơ sở dữ liệu mà AWS cung cấp.

Cơ sở Dữ liệu Nguồn (Source Databases)

Có thể là cơ sở dữ liệu tại chỗ, EC2-based (Oracle, SQL Server, MySQL, MongoDB, v.v.), cơ sở dữ liệu Azure, hoặc RDS/Aurora/S3/DocumentDB.

Cơ sở Dữ liệu Đích (Target Databases)

Bao gồm các cơ sở dữ liệu tại chỗ/EC2-based, Amazon RDS (bao gồm cả Aurora), Redshift, DynamoDB, Amazon S3, Kinesis Data Streams, Apache Kafka, DocumentDB, Neptune, Redis và Babelfish.

Triển khai Multi-AZ và Tính năng Bổ sung

Triển khai Multi-AZ cho DMS

DMS hỗ trợ triển khai Multi-AZ (Đa Vùng Sẵn sàng). Điều này có nghĩa là bạn có một Replication Instance ở một AZ và một bản sao (replica) đồng bộ ở một AZ khác làm bản sao dự phòng (standby).

  • Cung cấp khả năng phục hồi trước lỗi AZ cụ thể.
  • Đảm bảo dữ liệu dư thừa và loại bỏ hiện tượng đóng băng I/O.
  • Giảm thiểu các đột biến độ trễ (latency spikes).

TÓM TẮT CHÍNH

  • AWS Database Migration Service (DMS) cho phép di chuyển nhanh chóng, an toàn và có khả năng phục hồi từ tại chỗ sang AWS, trong khi cơ sở dữ liệu nguồn vẫn sẵn sàng.
  • DMS hỗ trợ cả di chuyển đồng nhất (homogeneous) và dị biệt (heterogeneous), với nhân bản liên tục thông qua Change Data Capture (CDC).
  • Một EC2 instance chạy phần mềm DMS thực hiện các tác vụ nhân bản.
  • AWS Schema Conversion Tool (SCT) là bắt buộc khi di chuyển giữa các công cụ cơ sở dữ liệu khác nhau (dị biệt).
  • DMS hỗ trợ triển khai Multi-AZ để đạt được tính sẵn sàng cao.
