Egress-Only Internet Gateway (EOIGW)

Giải Pháp Cho Kết Nối IPv6 Ra Bên Ngoài An Toàn

1. Giới Thiệu về Egress-Only Internet Gateway

Egress-Only Internet Gateways được sử dụng ĐỘC QUYỀN cho lưu lượng IPv6.

Chúng tương tự như NAT Gateways, nhưng trong khi NAT Gateways dành cho IPv4 thì EOIGW được thiết kế riêng cho IPv6.

Mục Đích Cốt Lõi

  • Cho phép Instance thiết lập kết nối ra bên ngoài (outbound) qua IPv6.
  • NGĂN CẢN Internet khởi tạo bất kỳ kết nối IPv6 nào vào Instance của bạn.

Lưu ý: Để kích hoạt chức năng này, bạn phải cập nhật Route Tables (Bảng định tuyến) tương ứng.

2. Kịch Bản So Sánh Subnet Công Cộng & Riêng Tư

Subnet Công Cộng

EC2 Instance có thể truy cập Internet qua IPv4 và IPv6 thông qua Internet Gateway (IGW).

Internet CÓ THỂ khởi tạo kết nối vào Instance (Inbound & Outbound).

Route Table: ::/0 -> IGW

Subnet Riêng Tư (An Toàn)

EC2 Instance cần truy cập Internet (ví dụ: cập nhật) mà vẫn ngăn chặn truy cập từ bên ngoài.

Internet KHÔNG THỂ khởi tạo kết nối vào (Chỉ Outbound).

Route Table: ::/0 -> Egress-Only IGW

3. Tổng Quan Định Tuyến (Routing) Chi Tiết

Dành cho Subnet Riêng Tư (Private Subnet)

Lưu lượng IPv4

Để truy cập Internet (Outbound), Instance định tuyến qua NAT Gateway.

Route Table: Lưu lượng 0.0.0.0/0 (tất cả) được gửi đến NAT Gateway.

Lưu lượng IPv6

Để truy cập Internet (Outbound), Instance định tuyến qua Egress-Only Internet Gateway.

Route Table: Lưu lượng ::/0 (tất cả) được gửi đến Egress-Only Internet Gateway.

TÓM TẮT CHỨC NĂNG GATEWAY

Gateway Giao thức Chức năng Vị trí áp dụng
Internet Gateway (IGW) IPv4 & IPv6 Cho phép Inbound & Outbound (truy cập hoàn toàn) Public Subnet
NAT Gateway IPv4 Chỉ Chỉ cho phép Outbound IPv4 Private Subnet
Egress-Only Internet Gateway IPv6 Chỉ Chỉ cho phép Outbound IPv6 (Ngăn chặn Inbound) Private Subnet

KEY TAKEAWAYS

  • Độc Quyền: EOIGW chỉ dùng cho lưu lượng IPv6 và cho phép kết nối outbound từ VPC.
  • Bảo Mật: Nó ngăn chặn Internet khởi tạo kết nối IPv6 inbound vào Instance (khác với IGW).
  • Routing: Route Tables của Private Subnet phải được cập nhật để chuyển lưu lượng IPv6 (::/0) qua EOIGW.
  • Phân Biệt: Nắm vững sự khác biệt giữa IGW (Public/Full Access), NAT Gateway (Private/IPv4 Outbound), và EOIGW (Private/IPv6 Outbound) là thiết yếu.
